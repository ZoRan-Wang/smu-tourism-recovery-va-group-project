Prototype: Clustering Analysis

Monthly tourism market states in Singapore’s recovery

1. Introduction

This prototype follows the same broad storytelling logic as the reference clustering page from Habari Tanzania: introduce the analytical objective, show the code that prepares the data, fit the clustering model, generate the results directly from code, and then explain what those results mean.

The difference is methodological. The Habari Tanzania example works with transformed categorical indicators for latent classes, while this project uses continuous monthly tourism indicators. For that reason, this page uses standardized k-means clustering at the monthly level.

The analytical question is:

Can the monthly tourism observations be grouped into a few interpretable market states defined by visitor arrivals, China’s market share, hotel occupancy, and length of stay?

All tables and figures below are generated directly from the code chunks in this document.

4. Prepare The Clustering Dataset

Introduction

The clustering unit must be the month. Following the project plan, the feature set contains:

  1. visitor_arrivals
  2. china_share
  3. hotel_occ
  4. avg_stay_monthly_capped

The raw avg_stay_monthly is kept for interpretation, but the capped version is used for modeling to reduce the influence of extreme pandemic months.

Code

feature_vars <- c(
  "visitor_arrivals",

  "hotel_occ",

5. Determine The Number Of Clusters

Introduction

The draft suggested starting with k = 3, but that decision still needs evidence. To evaluate the candidate cluster counts, the page computes:

  1. the total within-cluster sum of squares for the elbow pattern, and
  2. the mean silhouette score for separation quality.

Code

candidate_k <- 2:6
distance_matrix <- dist(cluster_matrix)

model_metrics <- bind_rows(lapply(candidate_k, function(k) {
  set.seed(42)
  fit <- kmeans(cluster_matrix, centers = k, nstart = 25)
  sil <- silhouette(fit$cluster, distance_matrix)

  data.frame(
    k = k,
    tot_withinss = fit$tot.withinss,
    silhouette = mean(sil[, "sil_width"], na.rm = TRUE)
  )
}))

elbow_plot <- ggplot(model_metrics, aes(x = k, y = tot_withinss)) +
  geom_line(color = "#0f3d3e", linewidth = 1.1) +
  geom_point(color = "#d86f45", size = 2.6) +
  scale_x_continuous(breaks = candidate_k) +
  labs(
    x = "Number of clusters (k)",
    y = "Total within-cluster sum of squares"
  ) +
  theme_minimal(base_size = 12)

silhouette_plot <- ggplot(model_metrics, aes(x = factor(k), y = silhouette, fill = k == 3)) +
  geom_col(width = 0.68, show.legend = FALSE) +
  scale_fill_manual(values = c("TRUE" = "#0f6b5b", "FALSE" = "#c7d4cf")) +
  labs(
    x = "Number of clusters (k)",
    y = "Mean silhouette score"
  ) +
  theme_minimal(base_size = 12)

metrics_table <- model_metrics |>
  transmute(
    K = k,
    `Total within-cluster SS` = round(tot_withinss, 1),
    `Mean silhouette` = round(silhouette, 3)
  )

Result

Explanation

The highest silhouette score occurs at k = r model_metrics$k[which.max(model_metrics$silhouette)], with a mean silhouette of 0.665. However, the elbow pattern bends noticeably by k = 3, and the three-cluster solution gives a more useful market narrative than a simple two-way split. For that reason, the prototype keeps k = 3 as the final setting while acknowledging that the choice prioritises interpretability over the absolute best silhouette value.

6. Fit The Final K-Means Model

Introduction

With k = 3 selected, the next step is to fit the final k-means model, assign each month to a cluster, and then translate the technical clusters into market-state labels.

Code

selected_k <- 3
set.seed(42)
cluster_fit <- kmeans(cluster_matrix, centers = selected_k, nstart = 25)
pca_fit <- prcomp(cluster_matrix)

cluster_results <- cluster_input |>
  mutate(
    cluster_id = cluster_fit$cluster,
    pc1 = pca_fit$x[, 1],
    pc2 = pca_fit$x[, 2]
  )

cluster_profile_raw <- cluster_results |>
  group_by(cluster_id) |>
  summarise(
    months = n(),
    visitor_arrivals = mean(visitor_arrivals),
    china_share = mean(china_share),
    hotel_occ = mean(hotel_occ),
    avg_stay_monthly = mean(avg_stay_monthly),
    avg_stay_monthly_capped = mean(avg_stay_monthly_capped),
    .groups = "drop"
  )

label_map <- cluster_profile_raw |>
  arrange(visitor_arrivals) |>
  mutate(
    state_label = c(
      "Pandemic shock months",
      "Recovery transition months",
      "High-performance months"
    )
  ) |>
  select(cluster_id, state_label)

cluster_results <- cluster_results |>
  left_join(label_map, by = "cluster_id") |>
  mutate(
    state_label = factor(
      state_label,
      levels = c(
        "Pandemic shock months",
        "Recovery transition months",
        "High-performance months"
      )
    ),
    period = factor(period, levels = c("pre_covid", "covid_shock", "recovery"))
  )

cluster_profile <- cluster_profile_raw |>
  left_join(label_map, by = "cluster_id") |>
  mutate(
    state_label = factor(
      state_label,
      levels = c(
        "Pandemic shock months",
        "Recovery transition months",
        "High-performance months"
      )
    )
  ) |>
  arrange(state_label)

cluster_size_table <- cluster_results |>
  count(state_label, name = "Months")

center_table <- as.data.frame(cluster_fit$centers) |>
  mutate(cluster_id = as.integer(rownames(cluster_fit$centers))) |>
  left_join(label_map, by = "cluster_id") |>
  transmute(
    `Market state` = state_label,
    `Center: visitor arrivals` = round(visitor_arrivals, 3),
    `Center: China share` = round(china_share, 3),
    `Center: hotel occupancy` = round(hotel_occ, 3),
    `Center: stay length` = round(avg_stay_monthly_capped, 3)
  ) |>
  arrange(match(`Market state`, levels(cluster_results$state_label)))

The final model produces three clearly different groups. The largest group is the high-performance state with 82 months, while the pandemic shock and recovery transition states capture the disrupted and transitional months that differ strongly from normal market conditions.

The centroid table is shown on the standardized scale, so the signs matter more than the raw magnitudes. Positive values indicate above-average conditions on a feature, while negative values indicate below-average conditions.

7. Visualise And Interpret The Results

Introduction

The next step is to turn the cluster assignments into interpretable outputs: a reduced-dimension scatter plot, a business-friendly profile table, and a timeline showing when each market state appears.

Code

state_palette <- c(
  "Pandemic shock months" = "#b6405d",
  "Recovery transition months" = "#ef9f23",
  "High-performance months" = "#0f6b5b"
)

pca_plot <- ggplot(cluster_results, aes(x = pc1, y = pc2, color = state_label, shape = period)) +
  geom_point(size = 3.1, alpha = 0.88) +
  scale_color_manual(values = state_palette) +
  labs(
    x = "Principal component 1",
    y = "Principal component 2",
    color = "Market state",
    shape = "Period"
  ) +
  theme_minimal(base_size = 12)

timeline_plot <- ggplot(cluster_results, aes(x = date, y = state_label, color = state_label)) +
  geom_point(size = 2.8, alpha = 0.9) +
  scale_color_manual(values = state_palette) +
  labs(
    x = "Month",
    y = "Market state",
    color = "Market state"
  ) +
  theme_minimal(base_size = 12)

profile_table <- cluster_profile |>
  transmute(
    `Market state` = state_label,
    Months = months,
    `Mean visitor arrivals` = format(round(visitor_arrivals), big.mark = ","),
    `Mean China share` = sprintf("%.1f%%", 100 * china_share),
    `Mean hotel occupancy` = sprintf("%.1f%%", hotel_occ),
    `Mean stay (raw)` = round(avg_stay_monthly, 1),
    `Mean stay (capped for model)` = round(avg_stay_monthly_capped, 1)

rownames(period_state_view) <- NULL
period_state_view <- period_state_view |>
  select(state_label, pre_covid, covid_shock, recovery) |>
  transmute(
    `Market state` = state_label,
    `Pre-COVID` = pre_covid,
    `COVID shock` = covid_shock,
    Recovery = recovery
  )

Explanation

The three-state solution is substantively interpretable.

  • Pandemic shock months average only 17,788 arrivals, with mean hotel occupancy around 56.2%.
  • Recovery transition months average 159,791 arrivals, showing a partial rebound but not a full return to pre-pandemic market performance.
  • High-performance months average 1,367,457 arrivals and 83.3% hotel occupancy, making them the closest to normal or strong tourism conditions.

One important nuance is that China’s share can be high even when China’s absolute arrival count is still low. This happens because the share is relative to the whole market. During the most disrupted months, the overall tourism market shrank so sharply that China’s proportion could remain elevated despite low absolute volumes. That is why the prototype keeps both volume and share in the interpretation.

The period table also shows the temporal logic of the clusters:

  • the high-performance state contains all pre_covid months and most recovery months,
  • the pandemic shock state is concentrated entirely in the covid_shock phase,
  • and the recovery transition state bridges the end of the shock period and the earliest recovery months.

9. Conclusion

This prototype now follows a stricter reproducible pattern:

  1. introduce the analytical objective,
  2. show the code,
  3. let the code generate the result,
  4. and then explain the meaning of that result.

The main conclusion remains the same, but it is now supported more transparently: monthly tourism observations in Singapore do not form one uniform sequence. They separate into interpretable market states associated with visitor volume, China’s market share, hotel occupancy, and stay duration.
