Prototype: Clustering Analysis

Monthly tourism market states in Singapore’s recovery

1. Introduction

This prototype now follows a tighter analytical rhythm: one focused code block, one direct output, and one explanation. The goal is to keep the page readable and fully reproducible at the same time.

The research question is:

Can monthly tourism observations be grouped into a few interpretable market states defined by visitor arrivals, China’s market share, hotel occupancy, and length of stay?

The method remains k-means clustering on standardized monthly indicators, because this dataset is continuous and month-based rather than respondent-based.

7. Standardize The Modeling Variables

K-means depends on Euclidean distance, so the four variables must be standardized before clustering. Otherwise, visitor counts would dominate the distance calculation simply because they are measured on a much larger scale.

cluster_matrix <- scale(as.matrix(cluster_input[, feature_vars, drop = FALSE]))

standardization_check <- data.frame(
  Variable = colnames(cluster_matrix),
  Mean_after_scaling = round(colMeans(cluster_matrix), 3),
  SD_after_scaling = round(apply(cluster_matrix, 2, sd), 3)
)

knitr::kable(standardization_check, align = c("l", "r", "r"))
Variable Mean_after_scaling SD_after_scaling
visitor_arrivals visitor_arrivals 0 1
china_share china_share 0 1
hotel_occ hotel_occ 0 1
avg_stay_monthly_capped avg_stay_monthly_capped 0 1

The means are approximately zero and the standard deviations are approximately one, so the standardized feature matrix is ready for clustering.

8. Generate The Elbow Plot

The first diagnostic for cluster selection is the elbow method. It compares how the within-cluster sum of squares changes as k increases.

candidate_k <- 2:6
distance_matrix <- dist(cluster_matrix)

model_metrics <- bind_rows(lapply(candidate_k, function(k) {
  set.seed(42)
  fit <- kmeans(cluster_matrix, centers = k, nstart = 25)
  sil <- silhouette(fit$cluster, distance_matrix)

  data.frame(
    k = k,
    tot_withinss = fit$tot.withinss,
    silhouette = mean(sil[, "sil_width"], na.rm = TRUE)
  )
}))

ggplot(model_metrics, aes(x = k, y = tot_withinss)) +
  geom_line(color = "#0f3d3e", linewidth = 1.1) +
  geom_point(color = "#d86f45", size = 2.6) +
  scale_x_continuous(breaks = candidate_k) +
  labs(
    title = "Elbow Plot for Candidate Cluster Counts",
    x = "Number of clusters (k)",
    y = "Total within-cluster sum of squares"
  ) +
  theme_minimal(base_size = 12)

The elbow bends clearly by around k = 3, which suggests that moving beyond three clusters yields diminishing returns in compactness.

9. Generate The Silhouette Plot

The second diagnostic is the mean silhouette score, which measures how well-separated the clusters are.

ggplot(model_metrics, aes(x = factor(k), y = silhouette, fill = k == 3)) +
  geom_col(width = 0.68, show.legend = FALSE) +
  scale_fill_manual(values = c("TRUE" = "#0f6b5b", "FALSE" = "#c7d4cf")) +
  labs(
    title = "Mean Silhouette Score by Candidate k",
    x = "Number of clusters (k)",
    y = "Mean silhouette score"
  ) +
  theme_minimal(base_size = 12)

The silhouette score is highest at k = r model_metrics$k[which.max(model_metrics$silhouette)], but k = 3 remains a defensible choice because it creates a more meaningful market-state interpretation than a simple two-cluster split.

11. Fit The Final K-Means Model

Once k = 3 is selected, the final model can be estimated and the month-level cluster assignments can be generated.

selected_k <- 3
set.seed(42)
cluster_fit <- kmeans(cluster_matrix, centers = selected_k, nstart = 25)
pca_fit <- prcomp(cluster_matrix)

cluster_results <- cluster_input |>
  mutate(
    cluster_id = cluster_fit$cluster,
    pc1 = pca_fit$x[, 1],
    pc2 = pca_fit$x[, 2]
  )

cluster_profile_raw <- cluster_results |>
  group_by(cluster_id) |>
  summarise(
    months = n(),
    visitor_arrivals = mean(visitor_arrivals),
    china_share = mean(china_share),
    hotel_occ = mean(hotel_occ),
    avg_stay_monthly = mean(avg_stay_monthly),
    avg_stay_monthly_capped = mean(avg_stay_monthly_capped),
    .groups = "drop"
  )

label_map <- cluster_profile_raw |>
  arrange(visitor_arrivals) |>
  mutate(
    state_label = c(
      "Pandemic shock months",
      "Recovery transition months",
      "High-performance months"
    )
  ) |>
  select(cluster_id, state_label)

cluster_results <- cluster_results |>
  left_join(label_map, by = "cluster_id") |>
  mutate(
    state_label = factor(
      state_label,
      levels = c(
        "Pandemic shock months",
        "Recovery transition months",
        "High-performance months"
      )
    ),
    period = factor(period, levels = c("pre_covid", "covid_shock", "recovery"))
  )

cluster_size_table <- cluster_results |>
  count(state_label, name = "Months") |>
  arrange(state_label)

largest_state_name <- as.character(cluster_size_table$state_label[which.max(cluster_size_table$Months)])
largest_state_months <- cluster_size_table$Months[which.max(cluster_size_table$Months)]

knitr::kable(cluster_size_table, align = c("l", "r"))

The fitted model produces three interpretable groups. In the current dataset, the largest group is High-performance months with 82 months.
