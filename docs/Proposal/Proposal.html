Singapore Tourism Time-Series Visual Analytics Prototype

Project Proposal

Author

Group Project Starter

Published

March 29, 2026

1. Overview and Motivation

Singapore tourism recovery is best understood as a time-series comparison problem rather than a single-market recovery story. The new full dataset gives the project monthly visitor arrivals by country and by transport mode, which makes it possible to study how different markets recovered at different speeds and with different seasonal patterns.

This proposal builds on the updated project direction recommended by the instructor. Instead of centering the whole application on month-based clustering or tree-based occupancy classification, the final system will focus on three linked analytical modules: time series visual analysis, time series clustering, and time series forecasting.

2. Problem Statement

Tourism recovery discussions often stop at a single headline number such as total arrivals. That is not enough for a visual analytics project. A more useful question is how the recovery path differs across markets, transport modes, and time windows, and how those differences can be communicated in an interactive Shiny application.

The project therefore needs a common time-series workflow that lets users:

  1. compare multiple countries or transport series over time,
  2. identify groups of series with similar recovery trajectories, and
  3. forecast future arrivals for selected series.

Without this structure, the analysis remains fragmented and the application cannot support a coherent user journey.

3. Data and Scope

The project now uses the instructor-provided full tourism workbook, which contains monthly arrival series for Singapore and a large set of countries and transport modes. The main analytical window is the continuous monthly block from December 2016 to February 2026.

Main data characteristics

  • Monthly frequency for the core arrival series
  • Country-level arrival counts
  • Transport-mode splits such as Air, Sea, and Land
  • Strong seasonality across major markets
  • Very low missingness in the monthly arrival block

Core series for the final app

  • Visitor Arrivals
  • Visitor Arrivals: China
  • Visitor Arrivals: India
  • Visitor Arrivals: Malaysia
  • Visitor Arrivals: Australia
  • Visitor Arrivals: Japan
  • Visitor Arrivals: Philippines
  • Visitor Arrivals: United Kingdom
  • Visitor Arrivals: Germany
  • Visitor Arrivals: Air
  • Visitor Arrivals: Sea
  • Visitor Arrivals: Land

The project will treat these series as the common analytical base for all three modules.

4. Project Aim

The project aims to develop a visual analytics prototype that helps users understand Singapore tourism recovery through comparison, clustering, and forecasting.

The application should allow users to:

  1. compare the trajectories of selected countries and transport modes,
  2. cluster series with similar time-series behavior,
  3. forecast future arrivals for selected series, and
  4. move smoothly between exploration, explanation, and prediction.

5. Research Questions

The proposal is guided by four questions:

  1. How did selected source markets differ in recovery timing and seasonal behavior?
  2. Which country or transport series have similar monthly arrival trajectories?
  3. How strongly does the Chinese market behave relative to other key markets in the recovery period?
  4. Can the project forecast future arrivals with enough structure to support a useful Shiny interface?

6. Analytical Modules

The final application is organized into three modules.

6.1 Time Series Visual Analysis

This module is the entry point for the application. It will show monthly arrival trajectories for selected countries and transport modes, with options to switch between raw values, indexed values, and share-based comparisons.

The main purpose of this module is to answer questions such as:

  • Which countries recovered fastest?
  • How did China’s relative contribution change over time?
  • Did Air, Sea, and Land recover in the same way?

The final UI should expose controls for series selection, time window selection, and display mode switching. The output should combine a clear line chart, an indexed comparison chart, and a concise summary table.

6.2 Time Series Clustering

The clustering module is designed to compare series, not months. This is the key pivot in the project. The cluster unit will be a country series or transport series, and the goal will be to group series with similar monthly arrival patterns.

This is a better fit for the data and for the instructor feedback because it turns clustering into a trajectory comparison tool. For example, China may cluster with other markets that show similar disruption and recovery timing, while markets such as Malaysia or Australia may form a different recovery group.

The module will use:

  • a selected set of countries or transport series,
  • a normalization step such as indexed or z-score transformation,
  • a similarity measure suitable for time-series comparison, and
  • a clustering method that produces interpretable groups.

The output should include:

  1. a cluster membership table,
  2. a representative pattern plot,
  3. a cluster diagnostic graphic,
  4. a recovery position map,
  5. a short cluster insight block that explains the strongest rebound group and China’s placement, and
  6. a downloadable assignment table.

6.3 Time Series Forecasting

The forecasting module will complete the analytical workflow by estimating future values for selected arrival series. This gives the app a forward-looking component and makes the final Shiny system more than a descriptive dashboard.

The forecasting task will likely compare a simple baseline with at least one time-series model, then display forecast accuracy and predicted future trajectories.

This module should support:

  • a target series selector,
  • a training cutoff control,
  • a forecast horizon setting,
  • a model choice control, and
  • forecast download output.

7. UI Design and Storyboard

The UI design follows a three-module structure:

  1. Time Series Visual Analysis
  2. Time Series Clustering
  3. Time Series Forecasting

The storyboard will document the user journey from exploration to comparison to prediction. It will show which controls are exposed in each module, what output is generated, and how the user can move from one analytical question to the next.

The design principle is to keep each module focused:

  • the visual analysis module explains what changed,
  • the clustering module explains which series behave similarly, and
  • the forecasting module explains what is likely to happen next.

9. Expected Contribution

The final system will contribute a coherent time-series visual analytics workflow for Singapore tourism.

It will:

  1. compare key source markets and transport modes over time,
  2. group similar series into interpretable clusters, and
  3. provide a forecasting view that extends the analysis into the future.

Compared with the old project framing, this version is more consistent with the instructor guidance and more suitable for a polished final Shiny application.
