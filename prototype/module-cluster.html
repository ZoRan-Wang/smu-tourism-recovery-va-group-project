Objective

Prototype a Shiny module that groups monthly observations into interpretable tourism market states.

Data Contract

Input dataset: data/raw/Name your insight (4).xlsx

Required fields:

  1. Visitor Arrivals
  2. Visitor Arrivals: China
  3. Hotel Room Occupancy Rate
  4. Average Length of Stay
  5. date

Analytical Design

  1. Load the shared monthly tourism workbook.
  2. Derive a common feature table including total arrivals, China share, hotel occupancy, and capped length of stay.
  3. Filter data by selected period.
  4. Select feature set and optionally standardize features.
  5. Run k-means with user-defined k.
  6. Compute silhouette score.
  7. Return cluster assignments, profile summary, and scatter plot.

Implementation Files

  3. app/R/mod_cluster_server.R
  4. app/app.R

Quality Gates

  1. No NA rows in selected features before clustering.
  2. Error message if filtered sample is too small for chosen k.
  3. Stable color palette across outputs.
  4. Reproducible clustering with fixed seed.
